triage-cli inbox — layouts considered

Snapshotted during the 2026-05-07 brainstorming session. Layout A was chosen for the first iteration; B and C are kept here in case A doesn't land in practice.

Layout A · Vertical split, list always visible CHOSEN

Left ~40% list, right ~60% detail. Selecting a row updates the right pane live. Status icons (◉ >> selected · ✓ done · → triaging · ○ pending · ✗ failed). You see the queue and the report at the same time — switching context is one keystroke.

Layout B · List → drill-in detail screen

Two full-screen views. List dominates; Enter swaps to a detail view, ⌫/q returns. Lots of room for both views. Trade-off: queue awareness disappears while reading the report.

Layout C · List + tabbed detail

What the original mockup proposed ([1] Summary [2] Evidence [3] Timeline …). List on top, tab strip in the middle, one section at a time below. Trade-off: navigation cost for what's effectively six short scrollable sections.
