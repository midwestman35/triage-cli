# Implementation Notes

- Working in isolated worktree `/tmp/opencode/triage-cli-inbox-chat-revamp` on branch `inbox-chat-revamp` because the original checkout is on `main` and has unrelated local changes, including files this plan touches. This avoids overwriting user work.
- The implementation plan includes per-task commit steps, but I am not committing automatically because repository instructions say to commit only when explicitly requested. I will leave changes unstaged unless asked otherwise.
- `ChatEvent` uses `#[serde(tag = "kind")]`; the `ProviderError` variant also needs an error category named `kind` in Rust. Serde cannot serialize both the discriminator and payload with the same JSON key, so the payload field is serialized as `error_kind` while the Rust field remains `kind`.
- While implementing directory attach, I also made `chat::attach_file` collision-safe for repeated basenames in the same turn. Without this, recursive `/dir` could attach `a/error.log` and `b/error.log` and the second copy would overwrite the first under `attachments/turn-NNN/error.log`.
- The plan's `ChatEvent::EvidenceAttached` shape uses `EvidenceProvenance`, but paste provenance contains the paste body. To honor the spec's logging PII/content boundary, `ChatLogger` sanitizes paste evidence before writing JSONL (`body` becomes `[omitted from chat-events.log]`). Direct serde round-trips of `ChatEvent` still preserve the Rust data shape.
- Adding `Option<&dyn ChatPhaseReporter>` made `pipeline::followup_turn` exceed clippy's argument-count threshold. I kept the plan's explicit signature and added a targeted `#[allow(clippy::too_many_arguments)]` instead of creating an options struct, because this is an internal pipeline entry point and the plan explicitly calls out the signature shape.
- `cargo fmt --all` also reformatted one pre-existing line break in `triage-cli-rs/src/cli.rs`. I left it because the plan requires repository formatting checks to pass.
- Final review caught that provider errors were redacted for `ChatEvent::ProviderError` but returned unredacted to the parent loop, which writes a System turn. The error branch now returns the redacted message so both the JSONL event and conversation System turn respect the same boundary.
- A follow-up review caught that file attachment extracted text was redacted neither in `build_followup_message` nor in provider stamping. `build_followup_message` now redacts extracted attachment text before provider dispatch.
